UIView UIImageView

发表于 2016-07-22   |   分类于 iOS学习笔记   |  

UIView的常用属性及方法|UIImageView 3种填充属性的不同

UIView的常用属性

  1. frame

    在父视图中的位置(参照点是父view坐标系统)

  2. bounds

    该view在本坐标系统中的位置和大小（参照点是本地坐标系统）     
一般控件的bounds和frame.size的尺寸是一样的

  3. center

    控件的中心点坐标

  4. UIViewContentModel 

    UIViewContentModeScaleToFill      缩放内容到合适比例大小
UIViewContentModeScaleAspectFit   缩放内容到合适的大小，边界多余部分透明
UIViewContentModeScaleAspectFill  缩放内容填充到指定大小，边界多余的部分省略
UIViewContentModeRedraw           重绘视图边界
UIViewContentModeCenter           视图保持等比缩放
UIViewContentModeTop              视图顶部对齐
UIViewContentModeBottom           视图底部对齐
UIViewContentModeLeft             视图左侧对齐
UIViewContentModeRight            视图右侧对齐
UIViewContentModeTopLeft          视图左上角对齐
UIViewContentModeTopRight         视图右上角对齐
UIViewContentModeBottomLeft       视图左下角对齐
UIViewContentModeBottomRight      视图右下角对齐

  5. 小属性

    hidden  隐藏视图
alpha   透明度（0~1）
opaque  不透明度 （0~1）
userInteractionEnabled 交互
multipleTouchEnabled   支持多点触控
exclusiveTouch         决定当前视图是否处理出门事件的唯一对象

  6. etc

    subviews                     当前视图的所有子视图（获取子视图）
superview                    当前视图的父视图（获取父视图）
tag                          当前视图标签,NSInteger类型
transform                    视图仿射变换,CGAffineTransform类型
                             CGAffineTransformMakeScale(CGFloat sx, CGFloat sy)缩放比例
userInteractionEnabled       决定与用户交互事件是否从被忽略并从事件队列中移除
windows                      当前视图上的UIWindow类型,可获取所在窗口对象

  7. 常用控件

    UIButton 按钮
UILabel 文本标签
UITextField 文本输入框
UIImageView 图片显示
UIScrollView 滚动的控件
UITableView 表格表格
UICollectionView 九宫格
UIWebView 网页显示控件
UIAlertView 对话框(中间弹框)
UINavigationBar 导航条

UIView的常用方法

  1. 创建方法

    UIView *view = [[UIView alloc] initWithFrame:frame];    
frame = CGRectMake(origini.x, origin.y, size.width, size.height);

  2. 添加视图/插入视图

    [view addSubview:subview]
在view上添加subview视图
当在view上添加多个子视图时,先添加的在下面,后添加的在上面

    使用以下方法调整子控件在subview数组中的顺序

    // 将子控件view插入到subviews数组的index位置
- (void)insertSubview:(UIView *)view atIndex:(NSInteger)index;

// 将子控件subview1显示到子控件subview的下面
- (void)insertSubview:(UIView *)subview1 belowSubview:(UIView *)subview;

// 将子控件subview1显示到子控件subview的上面
- (void)insertSubview:(UIView *)subview1 aboveSubview:(UIView *)subview;

// 将子控件view放到数组的最后面，显示在最上面
- (void)bringSubviewToFront:(UIView *)view;

// 将子控件view放到数组的最前面，显示在最下面
- (void)sendSubviewToBack:(UIView *)view;

  3. 删除视图

    [subview removeFromSuperview]   
子视图从父视图上删除,注意不能通过父视图删除子视图,只能子视图将自己在父视图上删去

  4. 获取视图对象

    [view viewWithTag:t] 获取view的子视图中标识为t的子视图对象

UIImageView 3种填充属性的不同

  • 创建

    //创建UIImageView对象
UIImageView *image =[[UIImageView alloc] 
initWithframe:CGRectMake(0,0,100,100)];

//指定图片
[image setImage:[UIImage imageNamed:@"1.jpg"]];

//指定背景
image.backgroundColor = [UIColor blueColor];

//设置UIImageView的填充属性
image.contentMode = UIViewContentModeScaleAspectFit;

//添加到view
[self.view addSubview:image];

  • 三种属性

  • UIViewContentModeScaleToFill

    > 会按照UIImageView的宽高比来拉伸图片
> 直到让整个图片都填充UIImageView为止
> 因为是按照UIImageView的宽高比来拉伸,所以图片会变形

  • UIViewContentModeScaleAspectFit

    常常配合:clipsToBounds(将超出的图片部分裁剪掉)
> 会按照图片的宽高比来拉伸
 > 要求整张图片都必须在UIImageView的范围内
 > 并且宽度和高度其中一个必须和UIImageView一样
 > 居中显示

  • UIViewContentModeScaleAspectFill

    > 会按照图片的宽高比来拉伸
 > 要求整张图片必须填充UIImageView
 > 并且图片的宽度或者高度其中一个必须和UIImageView一样
 > 因为是按照图片的宽高比来拉伸, 所以图片不会变形

  • 规律

    > 但凡取值中包含Scale单词的,都会对图片进行拉伸(缩放);
 > 但凡取值中没有出现Scale单词的,都不会对图片进行拉伸;
 > 但凡取值中包含Aspect单词的, 都会按照图片的宽高比来拉伸.
