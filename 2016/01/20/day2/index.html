iOS编码规范

发表于 2016-01-20   |   分类于 iOS学习笔记   |  

基本的iOS编码规范

  • 代码列数最大为100列

Text Editing中的Page guide at column设为100

  • 声明类或方法时，注意空格、换行的使用

声明类或方法时，注意空格的使用，参数过多时可换行保持对齐

调用方法时也是如此，参数都写在一行或换行冒号对齐

  • 命名规则（驼峰式命名）

「类名」首字母大写，「方法」首字母小写，「方法中的参数」首字母小写

同时尽量让方法看起来像一句话，能够传达出方法的意思

取值方法前不要加前缀「get」

(void)chooseCardAtIndex:(NSUInteger)index;

变量名小写字母开头

int chosenButtonIndex = [self.cardButton indexOfObject:sender];

常量以小写字母k开头，后续首字母大写 

static const NSString* kLNRouteServiceString = @“http://218.60.144.97/abc”;
  • 注释

注释很重要，但除了开头的版权声明，尽可能把代码写的如同文档一样，让别人直接看代码就知道意思，写代码时别担心名字太长，相信Xcode的提示功能

  • 尽可能保证 .h文件的简洁性

尽可能保持 .h文件的简洁性，可以不公开的API就不要公开了，写在实现文件中即可

  • #import 和 #include 使用

Xcode 支持 Objective-C/C/C++ 混编，所以引用头文件时：

#import Objective-C/Objective-C++头文件（Objective-C++是Objective-C与C++混编的文件）

#include C/C++ 头文件
  • 实例变量声明

实例变量声明时变量名前面加下划线”_”，局部变量不用加。

如：

AGSTiledMapServiceLayer* _vecServiceLayer;

为了避免与系统变量、工程里其他变量的名字相同

  • 使用 「#pragma mark」

如#pragma mark - addView，可在路径处选择标签

从而 快速找到标签下面所有函数的方法
